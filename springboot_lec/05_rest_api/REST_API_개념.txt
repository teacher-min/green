REST API 개발

1. REpresentational State Transfer
2. 자원(Resource)의 표현 방식(Representation)에 의한 상태 전달(State Transfer)로 구성
3. 자원의 요청 및 응답에 관한 규칙을 정의하는 것



REST 의 구성 요소

1. 자원(Resource)
  1) 서버에 존재하는 데이터를 의미합니다.
  2) 해당 데이터를 찾기 위한 URL 을 의미합니다.
2. 행위(Verb)
  1) HTTP Method 를 활용합니다.
  2) HTTP Method | 의미                  | 예시
     ------------|-----------------------|---------------------------
     GET         | 조회(Read, SELECT)    | GET /users
                 |                       | GET /users/1
     POST        | 생성(Create, INSERT)  | POST /users
     PUT         | 수정(Update, UPDATE)  | PUT /users/1
     DELETE      | 삭제(Delete, DELETE)  | DELETE /users/1
3. 표현(Representation)
  1) 자원을 표현하는 방식을 의미합니다.
  2) JSON, XML, TEXT, CSS, RSS 등 데이터의 종류를 의미한다.
  3) 최근에 가장 많이 사용하는 것은 JSON 입니다.



REST API 설계 방식

1. 자원은 URI로 표시하고, 행위는 HTTP Method로 표시합니다.
2. URI의 계층 구분은 슬래시(/)로 하고, 마지막엔 슬래시(/)를 표시하지 않습니다.
3. 소문자만 사용하고, 밑줄(_) 대신 하이픈(-)을 사용합니다.
4. 특정 파일 포맷의 경우 확장자를 생략합니다. 대신 Accept 헤더에 지정합니다.
5. 동작이나 행위를 의미하는 단어를 사용하지 않습니다.
  /user/regist, /user/list, /user/detail, /user/modify, /user/remove 등을 지양합니다.
6. 가급적이면 명사를 사용합니다. 동사를 지양합니다.
7. 쿼리 스트링 방식이 아닌 경로 변수 방식을 사용합니다.
  1) 쿼리 스트링 : url?userId=1
  2) 경로 변수   : url/userId/1
